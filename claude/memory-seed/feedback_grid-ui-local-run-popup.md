`ui5 serve`/`fiori run` ile freestyle grid (sap.ui.table + TablePersonalizer) app lokal açılınca tarayıcı **SAP basic-auth (user&pass) popup'ı** atıyor; doğru şifre girilse de geçmiyor. 2026-06-10 ZSD001 fittings_se'de yaşandı. **İki kaynak (ikisi de proxy→canlı backend→401→native popup):**
1. **UI5 flexibility (LREP):** `/sap/bc/lrep/flex/settings` + `/data`. `manifest flexEnabled:false` 1.120'de TAM durdurmaz.
2. **TablePersonalizer variant servisi `ZSD000_UI_VARIANT_O2`:** `new ODataModel(VAR_SERVICE_URL)` instantiation'ı metadata çekip 401 atıyor (bu serviste sistem 401 dönüyor — yayında değil/yetki yok → zaten her zaman localStorage fallback).

**Why:** `fiori run` run-time proxy auth'u **tarayıcı popup'ı** (keychain "No credential found"). `FIORI_TOOLS_USER/PASSWORD` env yalnız **deploy** içindir (standards/03 §2.4.1), run-proxy'de çalışmaz. Mockserver sadece app OData'sını mock'lar; lrep/variant canlıya gider.

**How to apply:** Grid UI'ı backend'siz lokal çalıştırmak için: (1) `index.html` bootstrap'a **`data-sap-ui-flexibility-services="[]"`** (lrep çağrısı sıfırlanır). (2) Bu app'in TablePersonalizer.js kopyasında **`LOCAL_VARIANTS_ONLY=true`** → `_varModel()` throw → localStorage-only (canlı ODataModel kurulmaz, popup yok). (3) `start-mock`'u **`index.html`** açacak şekilde ayarla (FLP/flp.html ekstra flex çağrısı yapar). (4) Doğrulama: server log'unda `lrep|VARIANT|401` = 0. Tarayıcıda Ctrl+Shift+R (cached JS temizle). İlişkili: [[feedback_powershell-utf8-bom-trap]], standards/03 §2.4.1.

**AUTH BLOKLUYKEN CLIENT-SIDE UI REPRO (playwright workaround, 2026-06-21):** Lokal-run proxy auth'u bloklayıp (keyring/@zowe yok, $metadata pending, env-creds yok) canlı OData GELMİYORSA ama **client-side davranışı** (autofit/yatay-scroll/formatter/guard) test etmen gerekiyorsa → backend'i bekleme: playwright `eval` ile app'in **JSONModel'ine sahte satır ENJEKTE et** (ör. `oModel.setProperty("/pool", [...30 geniş satır]); oModel.refresh()`), gerçek `autoResizeColumn`/render yolu çalışır, scrollLeft/getBoundingClientRect'i SAYIYLA ölç. Mekanizma (autofit→scroll-jump→reset) JSON vs OData'dan bağımsız → birebir aynı. ZSD001 shipment hscroll-fix böyle kanıtlandı (jump 0→898 ölçüldü, fix sonrası sabit 0). NOT: bu mekanizma-teyidi; gerçek-OData son-smoke yine kullanıcı authenticated ortamında. İlişkili: [[feedback_playwright-ui5-firepress-model-api]], [[feedback_fe-merge-key-padding-ve-playwright-runtime-tespit]].

**İKİNCİ TUZAK — popup ISRARLA geliyor ama lrep YOK = HESAP KİLİDİ (2026-06-11, ZSD001, ~1 saat kayıp):** Flex fix uygulandıktan sonra log'da yalnız app `$metadata` 401'i tekrar ediyor + popup geri geliyorsa → sebep lokal/proxy DEĞİL, **SAP kullanıcısı kilitli** (tekrarlı başarısız logon sayacı doldu — popup denemelerinin kendisi kilitler). **DAHA FAZLA DENEME YAPMA** (kiliti uzatır). Teşhis: MCP `adt_get` (herhangi obje, metadata-only) — aynı kullanıcı (<SAP_USER>) **401** alıyorsa kesin hesap. **DİKKAT (2026-06-12 inceltme):** `adt_get` **200** dönmesi tarayıcı logon'unun açık olduğunu GARANTİ ETMEZ — ADT/MCP önbellekli oturum/SSO cookie kullanıyor olabilir, oysa tarayıcı basic-auth yeniden kimlik doğrulayıp kilitli sayaca takılır. ZSD001 fittings_se'de tam bu yaşandı: adt_get 200 verdi ama tarayıcı popup'ı geçmedi → sebep yine hesap kilidiydi (kullanıcı SU01 ile açınca geldi). Yani 200'e bakıp "kilitli değil" deme; popup ısrarlıysa + teknik tuzaklar kapalıysa (flexibility-services=[] + LOCAL_VARIANTS_ONLY) → kilidi SU01'de doğrula. Çözüm: Basis SU01 unlock 🔓 (şifre dolduysa reset + `.conn_adt` güncelle). **Per-app reçete kalıcı yazıldı:** `ERP/SD/<PKG>/ui/<app>/RUN.md` (ZSD001: order_app_rap/RUN.md) — başlatma komutu + iki-tuzak karar ağacı; lokal UI başlatmadan ÖNCE bunu oku. Playbook: ui-freestyle-odata-v2.md §0 madde 9. ZSD001 çalışan komut: `npm run start-noflp` (index.html, flp.html değil).
